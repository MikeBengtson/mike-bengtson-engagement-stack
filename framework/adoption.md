# Adopting Engagement Stack

1. Fork or copy the repository.
2. Delete `examples/` if you do not want sample content.
3. Fill `profile/`, `cv/`, `portfolio/`, `evidence/`, and `engagements/`.
4. Keep `framework/` unless you intentionally remove framework metadata.
5. Use `prompts/populate-engagement-stack.md` for an agent-led setup interview.
6. Use `submissions/` to generate platform-specific material.
7. Use `maintenance/` to keep the stack current.

## Public-First, Private-Overlay Ready

The public repo should be safe to share. Sensitive fields can be kept out of public Markdown/YAML or maintained in a private overlay repo.
