# Adopting Engagement Stack

1. Fork or copy the repository.
2. Fill `profile/`, `cv/`, `portfolio/`, `evidence/`, and `engagements/`.
3. Keep `framework/` unless you intentionally remove framework metadata.
4. Use `prompts/populate-engagement-stack.md` for an agent-led setup interview.
5. Use `submissions/` to generate platform-specific material.
6. Use `maintenance/` to keep the stack current.

## Public-First, Private-Overlay Ready

The public repo should be safe to share. Sensitive fields can be kept out of public Markdown/YAML or maintained in a private overlay repo.
