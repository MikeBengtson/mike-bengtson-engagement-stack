# Social 2.0

## Summary

**Social 2.0** was a unified Java microservices platform at Amobee that consolidated the Facebook, Twitter, Snapchat, and Pinterest advertising APIs with internal data and state behind a single surface. Mike designed the platform and personally built its multi-identity OAuth, ad-hoc entity-grouping, and front-end orchestration services.

## Role

**Designer / lead** — within Amobee's Social Channels team (2016–2021). Designed the unified microservices platform and personally implemented core services (multi-identity OAuth, entity-grouping, front-end orchestration). Also led the founding Snapchat-channel "tiger team" and migrated the org off a Vagrant/Bash/Fabric toolchain to a Kubernetes DevOps solution. Evidence level: **built** (designer + hands-on) and **led**.

## What This Demonstrates

- Designing a unifying microservices platform across four disparate third-party ad APIs (Facebook, Twitter, Snapchat, Pinterest) plus internal data/state.
- Hands-on implementation of hard cross-cutting services: multi-identity OAuth, ad-hoc entity-grouping, and front-end orchestration.
- Leading a founding "tiger team" (Snapchat channel) building Java/Spring Boot/PostgreSQL microservices.
- Driving a DevOps modernization — replacing Vagrant/Bash/Fabric with Kubernetes — that cut configuration overhead and deployment errors.
